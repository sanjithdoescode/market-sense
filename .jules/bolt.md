## 2024-07-19 - Safe Application of Mongoose .lean()
**Learning:** While Mongoose's `.lean()` is an excellent performance optimization for read-only queries (reducing memory footprint and hydration time), applying it blindly to generic repository methods or inside deletion workflows is dangerous. `.lean()` strips methods like `.save()` and `.deleteOne()` and removes virtual getters (like `.id`). Fortunately, this project's `responseFormatter.js` safely normalizes Mongoose documents and POJOs via its `toPlain` and `toId` helpers.
**Action:** When optimizing Mongoose queries, trace the caller's usage. Only apply `.lean()` when you are certain the caller only reads the data and does not rely on Mongoose document methods or virtual getters that haven't been safely abstracted. Do not apply it to objects being retrieved for subsequent deletion logic.
